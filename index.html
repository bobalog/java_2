🧮 연산자 기본
개요
자바 연산자 총 43개시험출제
자바에서 제공하는 연산자 수: 43개
식 = 연산자 + 피연산자로 구성
종류: 산술식, 관계식, 논리식 등

❌ $ — 연산자 아님! (식별자 문자)
우선순위·결합
연산자 우선순위 대원칙핵심
높음 → 낮음 순서:
단항(!, 전위증감, cast) → 산술(*,/,%) → 산술(+,-) → 이동(shift) → 관계(>,<) → 항등(==,!=) → 비트AND(&) → 비트XOR(^) → 비트OR(|) → 논리AND(&&) → 논리OR(||) → 삼항(?:) → 배정(=)

우측 결합: ++(전위증감), 배정문(=), 삼항(?:), cast
⚖️ 관계·논리 연산자
관계 연산자
관계 연산자 핵심 규칙시험출제
관계식 결과: 항상 true 또는 false
우선순위: 산술 > 관계 > 논리 순
(산술이 먼저, 관계 비교 다음, 논리 결합 마지막)

항등 연산자(==, !=)는 비교 연산자보다 우선순위 낮음

❌ "관계 연산자가 산술보다 우선순위 높다" — 틀린 말!
특수 실수값
실수 연산의 특수값시험출제
양수 / 0.0 → Infinity (무한대)
0.0 / 0.0 → NaN (Not a Number)

NaN과 Infinity는 구분해야 해!
NaN == NaN → false (NaN은 자기 자신과도 같지 않음)
🔢 비트 연산자
비트 연산
비트 연산자 완전 정리시험출제
연산자이름규칙예시 (7₁₀ op 2₁₀)
&AND둘 다 1이면 1111 & 010 = 010 = 2
|OR하나라도 1이면 1111 | 010 = 111 = 7
^XOR서로 다르면 1111 ^ 010 = 101 = 5 ✅
~NOT비트 반전~7 = -8
<<왼쪽 이동×2 효과7 << 1 = 14
>>오른쪽 이동÷2, 부호 유지7 >> 1 = 3
>>>부호없는 오른쪽÷2, 항상 0 채움(자바 특유 연산자)
🔄 전위 vs 후위 증감
증감 연산자
전위(prefix) vs 후위(postfix)핵심
전위(++i): 먼저 증가 → 증가된 값으로 연산
후위(i++): 현재 값으로 연산 → 나중에 증가
int i = 5;
int a = ++i; // a=6, i=6
int b = i++; // b=6, i=7
⚠️ "후위가 전위보다 빠르다" — 틀린 말! 속도 차이는 없어.
🔃 형 변환
확장 변환
묵시적 형 변환 (Widening)핵심
작은 형 → 큰 형: 자동 변환
정보 손실 없음 (작은 컵 → 큰 컵으로 옮기는 느낌)

byte → short → int → long → float → double
char → int
✅ int → double → int 는 정보 손실 없어!
⚠️ long → double → long 은 정밀도 손실 가능!
협소화 변환
명시적 형 변환 (Narrowing)시험출제
큰 형 → 작은 형: 명시적 캐스팅 필요
정보 손실 발생 가능!

float → long ← 협소화! (실수 → 정수, 소수 잘림)
int i; float f; long l;
l = f + i; // 컴파일 에러!
오토박싱
Boxing / Unboxing핵심
Boxing: 기본형 → 참조형(래퍼 클래스) 자동 변환
Unboxing: 참조형 → 기본형 자동 변환

🔧 필드와 메소드
정적 vs 인스턴스
static 변수의 특성핵심

정적 변수 (static)

  • 클래스 전체가 공유
  • 메모리 1개만 존재
  • 클래스명으로 접근

인스턴스 변수

  • 객체마다 별도 존재
  • new 할 때마다 생성
  • 객체명으로 접근
필드 자격자
필드에 쓰는 자격자시험출제
final — 값 변경 불가 (상수)
volatile — 상수 전파 등 최적화 방지 (다른 곳에서 값 바뀔 수 있음)
static — 클래스 전체 공유
transient — 직렬화 제외

❌ synchronized, native, abstract — 필드 자격자 아님!
메소드 시그니처
시그니처(Signature) 구성시험출제
시그니처 = 메소드 이름 + 매개변수 개수 + 매개변수 형

❌ 반환형은 시그니처에 포함되지 않아!
❌ 매개변수 이름도 시그니처 아님!

오버로딩 구분: 컴파일러가 시그니처로 판단

실 매개변수(actual) — 메소드 호출할 때 사용

생성자
생성자(Constructor) 완전 정리핵심

생성자의 특징

  • new 연산자로 객체 생성 시 자동 호출
  • 이름 = 클래스 이름과 동일
  • 반환형 없음 (void도 안 씀!)
  • 주 역할: 필드 초기화
  • this()로 다른 생성자 호출 가능

실행 순서

  • 1️⃣ 정적 초기화문 실행
  • 2️⃣ 초기화문 실행
  • 3️⃣ 생성자 실행
클래스 차원 → 객체 차원 → 생성자 순
